# 스케줄워크 — 개인정보 처리방침

**시행일자:** 2026년 5월 16일

김지태(이하 "개발자")는 모바일 애플리케이션 **"스케줄워크 (ScheduleWork)"**(이하 "앱") 이용자의 개인정보를 중요시하며, 「개인정보 보호법」 등 관련 법령을 준수합니다. 스케줄워크는 교대·근무 일정과 할 일을 관리하는 도구 앱입니다.

## 1. 처리하는 개인정보 항목 및 수집 방법

앱은 별도의 회원가입이 필요 없으며, 근무 일정 관리 기능을 위해 이용자가 직접 입력한 정보와 서비스 이용 과정에서 자동으로 생성되는 정보, 그리고 이용자가 피드백을 보낼 때 전송되는 정보가 있습니다.

### 1-1. 이용자가 직접 입력하는 정보 (기기 내에만 저장)

| 구분 | 항목 |
|---|---|
| 근무 일정 정보 | 근무 일정, 근무 유형, 회사·근무지 이름 |
| 할 일·메모 정보 | 할 일 항목, 일별 메모, 기분 기록 |
| 워크스페이스 설정 | 이용자가 구성한 워크스페이스·표시 설정 |

이 정보는 **기기 내 SwiftData에만 저장**되며, **외부 서버로 전송되지 않습니다**. iCloud(CloudKit) 동기화는 사용하지 않으므로, 이용자가 입력한 일정·메모 데이터는 해당 기기를 벗어나지 않습니다. 위젯 표시를 위해 동일 기기 내 App Group 영역에 일부 데이터가 공유되며, 이 역시 기기를 벗어나지 않습니다.

### 1-2. 자동 수집 정보

| 구분 | 항목 |
|---|---|
| 기기 식별자 | IDFV, IDFA(광고 추적 동의 시에만) |
| 기기·환경 | OS 종류·버전, 기기 모델, 앱 버전, 언어·지역 설정 |
| 위치 | IP 주소 기반 대략적 위치(국가/지역 수준) |
| 이용 기록 | 화면 조회, 기능 사용 이벤트 |
| 광고 데이터 | 광고 노출·클릭, SKAdNetwork 어트리뷰션 |
| 진단·충돌 데이터 | Firebase Crashlytics를 통해 자동 수집되는 비정상 종료 로그, 오류 스택, 기기 상태 및 성능 지표 |

### 1-3. 이용자가 피드백을 보낼 때 전송되는 정보

앱 내 피드백 기능을 통해 이용자가 의견을 보낼 경우, 아래 정보가 Google Firestore(Google Cloud) 서버로 전송·저장됩니다. 계정·로그인이 없으므로 이름·이메일 등 이용자를 식별하는 정보는 포함되지 않습니다.

| 구분 | 항목 |
|---|---|
| 피드백 내용 | 이용자가 직접 작성한 의견 텍스트 |
| 기기·앱 환경 | 기기 모델, iOS 버전, 앱 버전, 전송 시각(서버 타임스탬프) |

## 2. 개인정보의 처리 목적

- 앱의 핵심 기능 제공: 근무 일정·근무 유형·할 일·메모의 생성·편집·표시
- 위젯을 통한 일정 표시 (기기 내 데이터만 사용)
- 서비스 품질 개선 및 이용 통계 분석 (Firebase Analytics)
- 앱 안정성 진단 및 오류 수정 (Firebase Crashlytics)
- 광고 제공 및 광고 효과 측정 (Google AdMob, Apple SKAdNetwork)
- 광고 동의 관리 (Google User Messaging Platform)
- 이용자 피드백 접수 및 서비스 개선 (Google Firestore)
- 인앱 구매 처리 (Apple StoreKit)

## 4. 개인정보의 제3자 제공 및 처리 위탁

앱은 다음 수탁업체에 일부 자동 수집 정보 및 피드백 정보의 처리를 위탁합니다. **이용자가 입력한 일정·메모 데이터는 어떠한 제3자에게도 제공되지 않습니다.**

| 수탁업체 | 위탁 업무 | 위탁 항목 |
|---|---|---|
| Google LLC (Firebase Analytics) | 앱 사용 분석 | 기기 식별자, 이벤트 로그, IP |
| Google LLC (Firebase Crashlytics) | 충돌·진단 분석 | 비정상 종료 로그, 기기·앱 상태 |
| Google LLC (Firebase Firestore) | 이용자 피드백 저장 | 피드백 내용, 기기 모델, iOS·앱 버전, 타임스탬프 |
| Google LLC (AdMob) | 광고 송출 및 측정 | 광고 식별자, 기기 정보, IP, 광고 이력 |
| Google LLC (User Messaging Platform) | 광고 동의 관리(EU/EEA 등) | 동의 상태, 기기 식별자 |
| Apple Inc. | 앱 배포, 인앱 구매 처리, 광고 어트리뷰션(SKAdNetwork) | 어트리뷰션 정보, 구매 거래 정보 |

수탁업체는 한국 외 국가(주로 미국)에 위치한 서버에서 데이터를 처리할 수 있습니다.

## 5. 정보주체의 권리 및 행사 방법

이용자는 다음 권리를 언제든지 행사할 수 있습니다.

- **본인이 입력한 일정·메모 데이터 열람·수정·삭제**: 앱 내에서 직접 수정·삭제 가능
- **앱 전체 데이터 삭제**: 앱 삭제 시 기기에 저장된 모든 일정·메모 데이터가 제거됩니다
- **피드백 정보의 열람·삭제**: 전송한 피드백은 식별자가 없어 개별 특정이 어려우나, 전송 시각·내용을 알려주시면 확인 후 삭제 처리합니다
- **추적 동의 철회**: iOS [설정] → [개인정보 보호 및 보안] → [추적] → [스케줄워크] 스위치 끄기
- **광고 식별자 재설정**: iOS [설정] → [개인정보 보호 및 보안] → [Apple 광고]
- 수탁업체가 보관 중인 자동 수집 정보에 대한 **열람·삭제·처리정지** 요청은 아래 연락처로 보내 주십시오. 정당한 요청은 지체 없이 처리합니다.

## 6. 개인정보의 안전성 확보 조치

- 이용자가 입력한 일정·메모 데이터는 외부 서버에 전송되지 않으며, 기기 내에서만 처리됩니다
- iCloud(CloudKit) 동기화를 사용하지 않으므로, 입력 데이터가 클라우드에 동기화되지 않습니다
- 피드백 전송 및 제3자 SDK와의 통신은 모두 HTTPS(TLS) 암호화 채널을 사용합니다
- 피드백에는 계정·식별자가 포함되지 않아 발신자를 특정하지 않습니다
- 수집되는 식별자는 익명·가명 처리된 형태로만 활용됩니다

## 7. 사용자 문의·피드백 기능

앱에는 이용자가 개발자에게 의견을 보낼 수 있는 피드백 기능이 포함되어 있습니다. 피드백을 전송하면 작성한 내용과 기기 모델·iOS 버전·앱 버전·전송 시각이 Google Firestore에 저장되며, 개발자가 이를 열람해 서비스 개선에 활용합니다. 피드백 본문에 포함되는 개인정보는 이용자 본인이 결정하므로, 본인을 식별할 수 있는 정보(이름·연락처 등)는 꼭 필요한 경우에만 입력해 주십시오.
