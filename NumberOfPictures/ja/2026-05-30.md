# プライバシーポリシー — Number on Photos (写真に番号を入力 / 사진에 숫자 입력)

**施行日:** 2026年5月30日

## 2. 取得する個人情報の項目

本アプリはアカウント登録を必要とせず、ユーザーから直接個人情報を入力させることはありません。ユーザーが端末内に作成・保存する情報と、本アプリの利用過程で第三者 SDK・システムが自動的に収集する情報があります。

### 2-1. ユーザーが作成・保存する情報(端末内のみ)

- **写真** — ユーザーが選択した写真は、番号合成のために **端末内でのみ処理** され、**当方が管理するサーバーにアップロードされることはありません**。合成結果をカメラロール(写真アプリ)に保存するかどうかはユーザーが選択します。
- **作業履歴** — 番号合成作業の記録(作成日時、画像名(参照)、入力モード)。**端末内の SwiftData にのみ保存** され、外部に送信されません。

### 2-2. 自動的に取得される情報(第三者 SDK・システムが取得)

以下の情報は **当方が直接取得・保管するものではなく、本アプリに組み込まれた第三者 SDK が、それぞれの目的に従って自動的に取得・処理** します。当方はこれらのデータを個人を識別できる形では保持せず、集計・匿名化された結果のみを確認します。

| 区分 | 項目 | 取得主体 |
|---|---|---|
| 端末識別子 | IDFV | Firebase Analytics、Firebase Crashlytics、Google AdMob |
| 広告識別子 | IDFA(App Tracking Transparency により同意した場合のみ) | Google AdMob |
| 端末・環境情報 | OSバージョン、デバイスモデル、アプリバージョン、言語・地域設定 | Firebase Analytics、Firebase Crashlytics、Google AdMob |
| 大まかな位置 | IPアドレスから推定される国・地域レベルの位置 | Firebase Analytics、Google AdMob |
| 利用履歴 | アプリ内イベント、画面閲覧 | Firebase Analytics |
| 診断・クラッシュデータ | クラッシュログ、パフォーマンス指標 | Firebase Crashlytics |
| 広告データ | 広告表示・クリック | Google AdMob |
| 広告アトリビューション | アプリのインストール・コンバージョンのアトリビューション | Apple SKAdNetwork |

## 3. 利用目的

- アプリの中核機能の提供(写真への番号合成、カメラロールへの保存、作業履歴の管理)
- サービス品質改善および利用統計分析(Firebase Analytics)
- クラッシュ・不具合の診断(Firebase Crashlytics)
- 広告の配信および効果測定(Google AdMob、Apple SKAdNetwork)
- 広告・トラッキングに関する同意の取得および管理(Google User Messaging Platform)
- 法令上の義務の履行

## 4. 第三者提供および委託

**ユーザーが端末内に作成・保存する写真および作業履歴は、いかなる第三者にも提供されません。** 自動取得情報のみ、以下の事業者に処理を委託します。

| 委託先 | 委託業務 | 委託項目 |
|---|---|---|
| Google LLC(Firebase Analytics, Crashlytics) | アプリ利用分析、クラッシュ診断 | 端末識別子、イベントログ、クラッシュログ、IP |
| Google LLC(AdMob / UMP) | 広告配信・測定および広告・トラッキング同意の管理 | 広告識別子、端末情報、IP、広告履歴、同意状態 |
| Apple Inc. | アプリ配信、広告アトリビューション(SKAdNetwork) | キャンペーン アトリビューション情報 |

## 8. トラッキング・同意の撤回方法

- iOS 設定 → プライバシーとセキュリティ → トラッキング → Number on Photos をオフ
- アプリ内の Google UMP 同意画面 — 表示される同意画面で広告・トラッキングの選択を変更できます
- iOS 設定 → プライバシーとセキュリティ → Apple広告 → 広告識別子のリセット
- iOS 設定 → プライバシーとセキュリティ → 写真 → Number on Photos でアクセス権限を変更
- 本アプリをアンインストールすると、端末内のローカルデータ(作業履歴、設定など)はすべて削除されます

## 9. 安全管理措置

- 写真および作業履歴は外部サーバーに送信されず、端末内でのみ処理・保存されます
- 第三者 SDK との通信はすべて HTTPS(TLS)暗号化により保護されます
- 取得される識別子は匿名化・仮名化された形でのみ利用されます
